# 2026-05-02 — Claude Code v2.1.126

## New Features
- `claude project purge [path]` — deletes all Claude Code state for a project (transcripts, tasks, file history, config entry). Supports `--dry-run`, `-y/--yes`, `-i/--interactive`, and `--all`. → **Affects:** Cheat Sheet (CLI subcommands table)
- `claude_code.skill_activated` OpenTelemetry event now fires for user-typed slash commands with new `invocation_trigger` attribute (`"user-slash"`, `"claude-proactive"`, `"nested-skill"`). → **Affects:** Claude Monitoring (OTEL events table)
- `claude_code.at_mention` OTEL log event added for `@`-mention resolution (v2.1.122). → **Affects:** Claude Monitoring (OTEL events table)

## Changed
- `--dangerously-skip-permissions` now bypasses prompts for writes to `.claude/`, `.git/`, `.vscode/`, shell config files, and other previously-protected paths (catastrophic removal commands still prompt). → **Affects:** Cheat Sheet CLI Flags table (update existing row)
- `ANTHROPIC_BEDROCK_SERVICE_TIER` env var added to select `default`, `flex`, or `priority` Bedrock service tier (v2.1.122). → **Affects:** Power Usage / Subprocess Sandboxing (narrow audience; low priority)
- `/model` picker now lists models from gateway's `/v1/models` when `ANTHROPIC_BASE_URL` points at a compatible gateway. → **Affects:** No playbook impact (gateway infra detail)

## Fixed
- Multiple stability fixes (stream idle timeout, image paste, OAuth, scrolling, Windows CJK, `Ctrl+L`, deferred tools in fork subagents). → **Affects:** No playbook impact
