# 2026-04-23 — Claude Code v2.1.118

## New Features
- Added vim visual mode (`v`) and visual-line mode (`V`) with selection, operators, and visual feedback → **Affects:** No playbook impact (editor UX, not documented)
- Create and switch between named custom themes from `/theme`, or hand-edit JSON files in `~/.claude/themes/`; plugins can also ship themes via a `themes/` directory → **Affects:** Cheat Sheet (update `/theme` row)
- Hooks can now invoke MCP tools directly via `type: "mcp_tool"` → **Affects:** Hooks — Four Types collapsible (add new row)
- Added `DISABLE_UPDATES` env var to completely block all update paths including manual `claude update` — stricter than `DISABLE_AUTOUPDATER` → **Affects:** Power Usage — Subprocess Sandboxing (add env var row)
- WSL on Windows can now inherit Windows-side managed settings via the `wslInheritsWindowsSettings` policy key → **Affects:** No playbook impact (Windows-specific policy key)
- Auto mode: include `"$defaults"` in `autoMode.allow`, `autoMode.soft_deny`, or `autoMode.environment` → **Affects:** No direct autoMode section in playbook; no action
- Added a "Don't ask again" option to the auto mode opt-in prompt → **Affects:** No playbook impact (UX prompt change)
- Added `claude plugin tag` to create release git tags for plugins with version validation → **Affects:** No playbook impact (plugin dev workflow)
- `--continue`/`--resume` now find sessions that added the current directory via `/add-dir` → **Affects:** No action needed (minor behavioral improvement, already documented)
- `/color` now syncs the session accent color to claude.ai/code when Remote Control is connected → **Affects:** No playbook impact
